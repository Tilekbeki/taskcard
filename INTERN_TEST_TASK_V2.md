Тестовое задание — React стажёр

Ожидаемое время выполнения: 1 час
Стек: React, TypeScript, CSS


Задача

Создай компонент карточки задачи (TaskCard) — UI-блок, который отображает информацию о задаче и позволяет менять её статус.
Компонент TaskCard должен принимать задачу через props.


Типы (обязательно использовать)

type TaskStatus = "todo" | "in-progress" | "done";
type TaskPriority = "low" | "medium" | "high";

interface Task {
  title: string;
  description: string;
  priority: TaskPriority;
  status: TaskStatus;
}

interface TaskCardProps {
  task: Task;
}


Что должно быть в карточке

- Заголовок задачи
- Описание задачи
- Приоритет — цветной бейдж (high — красный, medium — оранжевый, low — зелёный)
- Статус — цветной бейдж (todo, in-progress, done)
- Кнопка "Next status" — переключает статус по кругу: todo → in-progress → done → todo
- Если статус "done" — кнопка "Next status" должна быть заблокирована (disabled)


Пример данных

const task: Task = {
  title: "Настроить авторизацию",
  description: "Реализовать вход по email и паролю с валидацией",
  priority: "high",
  status: "todo",
};


Технические требования

- React — функциональный компонент
- TypeScript — использовать типы выше, никаких any
- Статус должен храниться в состоянии компонента (useState)
- Стили — любые (CSS, inline). Стили могут быть простыми — оценивается аккуратность, а не дизайн


Как сдать

1. Залить код на GitHub в публичный репозиторий
2. Прислать ссылку на репозиторий


Что оцениваем

Логика — правильно ли работает смена статуса, заблокирована ли кнопка при done
TypeScript — используются готовые типы, есть интерфейс для пропсов, нет any
Стили — цвета приоритетов, читаемость
Чистота кода — понятные имена, нет лишнего
